# AllAI Mobile App Docs

This folder is the project memory for AllAI Mobile App.

Use these files as the stable reference before implementation work:

- `PROJECT_SPEC.md` - full combined product spec.
- `ACTIVE_SPRINT.md` - current phase, role assignments, and next coordinator actions.
- `PHASE_0_ROLE_OUTPUTS.md` - merged outputs from role threads during Phase 0.
- `PHASE_1_SCAFFOLD_RESULT.md` - verified Flutter scaffold result and local build notes.
- `PHASE_2_ROLE_ASSIGNMENTS.md` - current design-system/static-UI task distribution.
- `PHASE_2_EXECUTION_BRIEF.md` - execution brief for the current static UI implementation pass.
- `PRODUCT_REQUIREMENTS.md` - product vision, MVP scope, priorities, acceptance.
- `MOBILE_ARCHITECTURE.md` - recommended stack, app architecture, module boundaries.
- `UX_SCREENS.md` - screen map, navigation, mobile UX states.
- `DATA_AND_API.md` - domain model, API contracts, storage, job lifecycle.
- `QA_RELEASE.md` - acceptance checklist, smoke tests, release risks.
- `TASK_THREADS.md` - Codex task-chat roles and handoff model.
- `REPO_SETUP.md` - repository structure, git, CI, first commit plan.
- `DEVELOPMENT_PLAN.md` - staged roadmap from empty repository to final product.
- `FLUTTER_SCAFFOLD_PLAN.md` - executable Phase 1 scaffold plan for Flutter.
- `DECISIONS_AND_OPEN_QUESTIONS.md` - current decisions and unresolved questions.
- `SOURCE_SNAPSHOT.md` - external product pages and observed product signals.

Snapshot date: 2026-07-02.
